LOGIN

UMLUseCase
Projeto_Mold :: model_ENTREGA1 :: LOGIN

Description

Pré-condições: O cliente deve possuir uma conta registrada na Tireguard.

Pós-condições: A Tireguard verifica se a conta é um login válido.

Fluxo Principal

1 - Cliente fornece seus dados para o sistema

2 - Sistema recebe os dados fornecidos e os valida

3 - Sistema libera o acesso no sistema em caso positivo de validação.

Fluxo Alternativo

1 - Cliente fornece seus dados

2 - Sistema recebe os dados fornecidos e os valida

3- sistema recusa o acesso e informa que os dados devem ser adicionados novamente, pois estão incorretos

Fluxo de exceção:

1- cliente fornece os dados

2 - Sistema recebe os dados fornecidos e os valida

3- Sistema informa que não existe uma conta criada para o login informado.
